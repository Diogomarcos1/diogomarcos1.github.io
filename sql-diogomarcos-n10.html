Introdução ao Trabalho com SQL

SQL, ou Linguagem de Consulta Estruturada, é uma linguagem de programação projetada para gerir e manipular dados armazenados em bancos de dados relacionais. Ele fornece uma forma padrão de aceder, atualizar e gerir bancos de dados.

Esta linguagem é amplamente utilizada em todo o mundo devido à sua simplicidade e eficácia na realização de uma variedade de operações em bancos de dados. Com o SQL, é possível realizar tarefas como:

  1. Consultar Dados
  2. Atualizar Dados
  3. Gerir Esquema
  4. Gerir Permissões de Acesso

Em resumo, o SQL é uma ferramenta poderosa para trabalhar com bancos de dados relacionais, permitindo aos desenvolvedores e administradores de banco de dados realizar uma ampla gama de operações para armazenar, recuperar e gerir dados de forma eficiente.


Comandos Básicos em SQL

Comandos Básicos

SELECT

Usado para selecionar dados de uma tabela.

SELECT * FROM produtos;

Seleciona todos os registros da tabela 'produtos'.

FROM

Especifica a tabela da qual os dados serão selecionados.

SELECT * FROM clientes;

Seleciona todos os registros da tabela 'clientes'.

WHERE

Filtra os resultados com base em uma condição específica.

SELECT * FROM pedidos WHERE valor_total > 100;

Seleciona pedidos onde o valor total é superior a 100.

ORDER BY

Ordena os resultados em uma ordem específica (ascendente ou descendente).

SELECT * FROM vendas ORDER BY data DESC;

Seleciona vendas ordenadas pela data em ordem descendente.

AND

Combina duas ou mais condições em uma instrução SELECT.

SELECT * FROM clientes WHERE idade > 30 AND pais = 'Brasil';

Seleciona clientes com idade acima de 30 anos e que sejam do Brasil.

OR

Retorna registros se qualquer uma das condições especificadas for verdadeira.

SELECT * FROM produtos WHERE categoria = 'Eletrônicos' OR preco > 500;

Seleciona produtos na categoria 'Eletrônicos' ou com preço superior a 500.

Operadores e Funções

BETWEEN

Filtra os resultados para que estejam dentro de um intervalo específico.

SELECT * FROM vendas WHERE quantidade BETWEEN 100 AND 200;

Seleciona vendas onde a quantidade está entre 100 e 200.

NOT BETWEEN

Filtra os resultados para que estejam fora de um intervalo específico.

SELECT * FROM vendas WHERE quantidade NOT BETWEEN 100 AND 200;

Seleciona vendas onde a quantidade não está entre 100 e 200.

IN

Verifica se um valor está presente em uma lista de valores.

SELECT * FROM clientes WHERE país IN ('Brasil', 'Portugal', 'Argentina');

Seleciona clientes cujo país está entre 'Brasil', 'Portugal' ou 'Argentina'.

NOT IN

Verifica se um valor não está presente em uma lista de valores.

SELECT * FROM clientes WHERE país NOT IN ('Brasil', 'Portugal', 'Argentina');

Seleciona clientes cujo país não está entre 'Brasil', 'Portugal' ou 'Argentina'.

IS NULL

Verifica se um valor é nulo.

SELECT * FROM clientes WHERE email IS NULL;

Seleciona todos os registros da tabela 'clientes' onde o campo de e-mail está vazio ou é nulo (NULL).

IS NOT NULL

Verifica se um valor não é nulo.

SELECT * FROM clientes WHERE email IS NOT NULL;

Seleciona todos os registros da tabela 'clientes' onde o campo de e-mail não está vazio ou nulo (NULL).

LIKE

Procura por padrões em uma string.

SELECT * FROM clientes WHERE nome LIKE '%Silva%';

Seleciona clientes cujo nome contém a string 'Silva'.

Operadores Matemáticos

+, -, *, /, %, ||

SELECT preço * quantidade AS total FROM vendas;

Calcula o total de cada venda multiplicando o preço pela quantidade.

Operadores de Comparação

COUNT

Retorna o número de linhas que correspondem a uma condição especificada.

SELECT COUNT(*) AS total_clientes FROM clientes;

Conta o número de clientes na tabela 'clientes'.

MAX

Retorna o valor máximo de uma coluna.

SELECT MAX(idade) AS idade_maxima FROM clientes;

Retorna a idade máxima dos clientes na tabela 'clientes'.

MIN

Retorna o valor mínimo de uma coluna.

SELECT MIN(idade) AS idade_minima FROM clientes;

Retorna a idade mínima dos clientes na tabela 'clientes'.

AVG

Retorna a média dos valores de uma coluna.

SELECT AVG(idade) AS idade_media FROM clientes;

Retorna a idade média dos clientes na tabela 'clientes'.

SUM

Retorna a soma dos valores de uma coluna.

SELECT SUM(valor) AS total_vendas FROM vendas;

Retorna o total de vendas na tabela 'vendas'.

ORDER BY

Ordena os resultados por uma ou mais colunas.

SELECT * FROM clientes ORDER BY nome ASC;

Ordena os clientes por nome em ordem alfabética crescente.

GROUP BY

Agrupa os resultados por uma ou mais colunas.

SELECT país, COUNT(*) AS total_clientes FROM clientes GROUP BY país;

Agrupa os clientes por país e conta o número de clientes em cada país.

HAVING

Filtra os resultados após o agrupamento.

SELECT país, COUNT(*) AS total_clientes FROM clientes GROUP BY país HAVING COUNT(*) > 10;

Agrupa os clientes por país e conta o número de clientes em cada país. Seleciona apenas os países com mais de 10 clientes.

DISTINCT

Retorna apenas linhas únicas.

SELECT DISTINCT país FROM clientes;

Retorna apenas os países distintos na tabela 'clientes'.

SUBSELECT

Uma consulta dentro de outra consulta.

SELECT * FROM clientes WHERE país IN (SELECT país FROM vendas);

Seleciona todos os clientes cujos países estão presentes na tabela 'vendas'.


Comandos SQL

SELECT

Para consultar dados de uma tabela, usamos o comando SELECT. Por exemplo:

SELECT * FROM tabela

Isso retorna todos os dados da tabela "tabela".

Exemplo:

SELECT * FROM clientes

Isso retorna todos os dados da tabela "clientes".

INSERT INTO

Para inserir novos dados em uma tabela, utilizamos o comando INSERT INTO. Por exemplo:

INSERT INTO tabela (coluna1, coluna2, coluna3) VALUES (valor1, valor2, valor3)

Isso adiciona um novo registro à tabela "tabela" com os valores especificados nas colunas.

Exemplo:

INSERT INTO clientes (nome, idade, email) VALUES ('João', 30, 'joao@email.com')

Isso adiciona um novo cliente à tabela "clientes" com o nome "João", idade "30" e email "joao@email.com".

UPDATE

Para atualizar dados existentes em uma tabela, usamos o comando UPDATE. Por exemplo:

UPDATE tabela SET coluna = valor WHERE condição

Isso atualiza os dados na coluna especificada na tabela com base na condição fornecida.

Exemplo:

UPDATE clientes SET idade = 35 WHERE nome = 'João'

Isso atualiza a idade do cliente chamado "João" para 35 anos na tabela "clientes".

DELETE

Para excluir registros de uma tabela, usamos o comando DELETE. Por exemplo:

DELETE FROM tabela WHERE condição

Isso exclui os registros da tabela com base na condição especificada.

Exemplo:

DELETE FROM clientes WHERE idade > 60

Isso exclui todos os clientes com idade superior a 60 anos da tabela "clientes".

CREATE TABLE

Para criar uma nova tabela, utilizamos o comando CREATE TABLE. Por exemplo:

CREATE TABLE tabela (
        coluna1 tipo1,
        coluna2 tipo2,
        coluna3 tipo3
        )

Isso cria uma nova tabela chamada "tabela" com as colunas e tipos especificados.

Exemplo:

CREATE TABLE produtos (
        id INT PRIMARY KEY,
        nome VARCHAR(100),
        preco DECIMAL(10, 2)
        )

Isso cria uma nova tabela chamada "produtos" com as colunas "id", "nome" e "preco".

ALTER TABLE

O comando ALTER TABLE é utilizado para modificar a estrutura de uma tabela existente no banco de dados. Ele permite adicionar, modificar ou excluir colunas da tabela, além de adicionar ou remover restrições, como chaves primárias e estrangeiras.

Aqui está a sintaxe completa do comando ALTER TABLE:

ALTER TABLE nome_tabela
        ADD COLUMN nome_atributo_1 tipo_1 [{restrições}] 
          [{, nome_atributo_n  tipo_n [{restrições}]}]
          |
        MODIFY COLUMN nome_atributo_1 tipo_1 [{restrições}] 
          [{, nome_atributo_n  tipo_n [{restrições}]}]
          |
        DROP COLUMN nome_atributo_1 
          [{, nome_atributo_n }]
          |
        ADD CONSTRAINT nome_RI_1 def_RI_1
          [{, nome_RI_n def_RI_n}]
          |
        DROP CONSTRAINT nome_RI_1
          [{, nome_RI_n}]
          |
        [ADD|DROP] [PRIMARY KEY ...|FOREIGN KEY ...];

Agora vamos ver exemplos de cada variante de uso do comando ALTER TABLE:

Adicionar uma nova coluna:

ALTER TABLE clientes
        ADD COLUMN telefone VARCHAR(15);

Este comando adiciona uma nova coluna chamada "telefone" à tabela "clientes", permitindo armazenar os números de telefone dos clientes.

Modificar o tipo de uma coluna:

ALTER TABLE clientes
        MODIFY COLUMN idade INT;

Este comando modifica o tipo de dados da coluna "idade" na tabela "clientes" para um tipo inteiro.

Remover uma coluna:

ALTER TABLE clientes
        DROP COLUMN email;

Este comando remove a coluna "email" da tabela "clientes", excluindo-a permanentemente da estrutura da tabela.

Adicionar uma restrição de chave estrangeira:

ALTER TABLE pedidos
        ADD CONSTRAINT fk_cliente_id
        FOREIGN KEY (cliente_id) REFERENCES clientes(id);

Este comando adiciona uma restrição de chave estrangeira na tabela "pedidos", relacionando a coluna "cliente_id" com a coluna "id" na tabela "clientes".

Remover uma restrição de chave estrangeira:

ALTER TABLE pedidos
        DROP CONSTRAINT fk_cliente_id;

Este comando remove a restrição de chave estrangeira chamada "fk_cliente_id" da tabela "pedidos".

Adicionar uma chave primária:

ALTER TABLE produtos
        ADD PRIMARY KEY (id);

Este comando adiciona uma chave primária à coluna "id" na tabela "produtos", garantindo que cada registro tenha um valor único para essa coluna.

Remover uma chave primária:

ALTER TABLE produtos
        DROP PRIMARY KEY;

Este comando remove a chave primária da tabela "produtos", se ela existir.

UNION

O operador UNION é usado para combinar o resultado de duas ou mais instruções SELECT em um único conjunto de resultados.

SELECT coluna1 FROM tabela1
        UNION
        SELECT coluna2 FROM tabela2;

Isso combina os resultados das consultas selecionando "coluna1" da "tabela1" e "coluna2" da "tabela2".

INNER JOIN

O INNER JOIN é usado para combinar linhas de duas ou mais tabelas com base em uma condição de junção especificada.

SELECT colunas
        FROM tabela1
        INNER JOIN tabela2 ON tabela1.coluna = tabela2.coluna;

Isso combina as linhas das tabelas "tabela1" e "tabela2" onde os valores das colunas correspondentes são iguais.


NATURAL JOIN

O NATURAL JOIN é usado para combinar linhas de duas tabelas com base em todas as colunas com o mesmo nome.

SELECT colunas
        FROM tabela1
        NATURAL JOIN tabela2;

Isso combina as linhas das tabelas "tabela1" e "tabela2" onde todas as colunas com o mesmo nome têm valores iguais.


LEFT JOIN

O LEFT JOIN (também conhecido como LEFT OUTER JOIN) retorna todas as linhas da tabela à esquerda e as linhas correspondentes da tabela à direita. Se não houver correspondência, retorna NULL nas colunas da tabela à direita.

SELECT colunas
        FROM tabela1
        LEFT JOIN tabela2 ON tabela1.coluna = tabela2.coluna;

Isso retorna todas as linhas de "tabela1" e as linhas correspondentes de "tabela2". Se não houver correspondência, as colunas de "tabela2" serão NULL.


RIGHT JOIN

O RIGHT JOIN (também conhecido como RIGHT OUTER JOIN) retorna todas as linhas da tabela à direita e as linhas correspondentes da tabela à esquerda. Se não houver correspondência, retorna NULL nas colunas da tabela à esquerda.

SELECT colunas
        FROM tabela1
        RIGHT JOIN tabela2 ON tabela1.coluna = tabela2.coluna;

Isso retorna todas as linhas de "tabela2" e as linhas correspondentes de "tabela1". Se não houver correspondência, as colunas de "tabela1" serão NULL.


FULL JOIN

O FULL JOIN (também conhecido como FULL OUTER JOIN) retorna todas as linhas quando há uma correspondência em uma das tabelas. Se não houver correspondência, as colunas da outra tabela serão NULL.

SELECT colunas
        FROM tabela1
        FULL JOIN tabela2 ON tabela1.coluna = tabela2.coluna;

Isso retorna todas as linhas de ambas as tabelas. Se não houver correspondência, as colunas da tabela sem correspondência serão NULL.

EXISTS

O EXISTS é usado para verificar a existência de qualquer registro em uma subconsulta.

SELECT colunas
        FROM tabela1
        WHERE EXISTS (SELECT * FROM tabela2 WHERE condição);

Isso retorna linhas da "tabela1" onde uma subconsulta para a "tabela2" retorna pelo menos um resultado.

LIMIT

O LIMIT é usado para limitar o número de registros retornados por uma consulta.

SELECT colunas
        FROM tabela
        LIMIT valor;

Isso limita o número de linhas retornadas pela consulta ao valor especificado.

GROUP BY

O GROUP BY é usado para agrupar registros que têm o mesmo valor em uma ou mais colunas.

SELECT colunas
        FROM tabela
        GROUP BY coluna;

Isso agrupa os registros da tabela com base nos valores da coluna especificada.

HAVING

O HAVING é usado em conjunto com o GROUP BY para filtrar registros agrupados.

SELECT colunas
        FROM tabela
        GROUP BY coluna
        HAVING condição;

Isso filtra os registros agrupados com base na condição especificada.


ÍNDICES

Índices em base de dados são estruturas que melhoram a velocidade de recuperação de dados. Eles funcionam como tabelas auxiliares que contêm chaves de busca (normalmente valores de colunas) e ponteiros para as linhas correspondentes em uma tabela. Quando você cria um índice em uma tabela, você está efetivamente pré-processando as consultas de busca, tornando-as mais eficientes.

Criar um Índice

Para criar um índice em uma tabela, você usa o comando CREATE INDEX. Por exemplo:

CREATE INDEX idx_nome_indice ON tabela (coluna);

Isso cria um índice chamado "idx_nome_indice" na tabela "tabela" para a coluna especificada.

Remover um Índice

Para remover um índice de uma tabela, você usa o comando DROP INDEX. Por exemplo:

DROP INDEX idx_nome_indice;

Isso remove o índice chamado "idx_nome_indice" da tabela.


VIEWS

Views em base de dados são consultas SQL armazenadas que podem ser tratadas como tabelas virtuais. Elas permitem que você armazene uma consulta complexa em forma de tabela virtual dentro do banco de dados. As views são úteis quando você tem consultas frequentes e complexas que precisam ser executadas repetidamente. Em vez de escrever a consulta repetidamente, você pode criar uma view e consultá-la sempre que precisar.

Criar uma View

Para criar uma view em um banco de dados, você usa o comando CREATE VIEW. Por exemplo:

CREATE VIEW nome_view AS
    SELECT colunas
    FROM tabela
    WHERE condição;

Isso cria uma view chamada "nome_view" que contém os resultados da consulta especificada.

Remover uma View

Para remover uma view de um banco de dados, você usa o comando DROP VIEW. Por exemplo:

DROP VIEW nome_view;

Isso remove a view chamada "nome_view" do banco de dados.
